---

## Request

Record a payment against a posted bill. This creates a payment record and updates the bill's amount due.

### Headers

<ParamField header="Authorization" type="string" required>
  JWT authentication token. Format: `jwt YOUR_TOKEN`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

### Path Parameters

<ParamField path="bill_id" type="string" required>
  The unique identifier of the bill
</ParamField>

### Body Parameters

<ParamField body="payment_date" type="string" required>
  Date of payment (YYYY-MM-DD)
</ParamField>

<ParamField body="payment_amount" type="number" required>
  Amount being paid
</ParamField>

<ParamField body="payment_account_id" type="string" required>
  Cash/bank account ID for the payment
</ParamField>

<ParamField body="payment_method" type="string">
  Payment method (e.g., `Check`, `ACH`, `Wire`, `Credit Card`)
</ParamField>

<ParamField body="reference" type="string">
  Payment reference number (e.g., check number)
</ParamField>

<Note>
  For more complex payment scenarios with multiple bills, use the [Bill Payments API](/api-reference/general-ledger/bill-payments/overview).
</Note>

## Accounting Impact

When a bill payment is recorded:
- **Debit**: Accounts Payable (reduces liability)
- **Credit**: Cash/Bank Account (reduces asset)
