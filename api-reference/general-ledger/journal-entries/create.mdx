## Request

Create a new journal entry. The entry is created in Draft status and must be submitted for approval before posting.

### Headers

<ParamField header="Authorization" type="string" required>
  JWT authentication token. Format: `jwt YOUR_TOKEN`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

### Body Parameters

<ParamField body="journal_entry_id" type="string" required>
  Unique identifier for the journal entry
</ParamField>

<ParamField body="tran_date" type="string" required>
  Transaction date (YYYY-MM-DD format)
</ParamField>

<ParamField body="posting_period_id" type="string" required>
  ID of the accounting period to post to
</ParamField>

<ParamField body="memo" type="string">
  Description of the journal entry
</ParamField>

<ParamField body="entry_type" type="string" default="Standard">
  Entry type: `Standard`, `Adjusting`, `Closing`, `Reversing`
</ParamField>

<ParamField body="currency" type="string" default="USD">
  ISO 4217 currency code
</ParamField>

<ParamField body="lines" type="array" required>
  Array of line items. Each line must have:
  - `line_number` (number): Sequential line number
  - `account_id` (string): GL account ID
  - `debit` (number): Debit amount (0 if credit)
  - `credit` (number): Credit amount (0 if debit)
  - `memo` (string, optional): Line description
</ParamField>

<Warning>
  The total debits must equal total credits. The API will reject unbalanced entries.
</Warning>
